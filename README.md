Создать приложение  для тестирования знаний разработчиков.
Приложение должно содержать три страницы (навигационная панель => HOME, QUESTIONS, ABOUT; header => название команды и логотип.). По дефолту открывается страница HOME.
На странице HOME необходимо вывести информацию об участниках команды, в виде карточек разработчика. В карточке должны быть следующие данные:
-Avatar (фото разработчика);
-Main info (возраст, рост, вес и т.д. Не менее 5ти параметров)
-Hobbie (2, 3 предложения);
	Реализовать возможность изменения данных (EDIT). Все данные разработчиков необходимо загружать с файла developers.json, все возможные модификации данных 
напрямую сохраняются в этот файл. 
Страница QUESTIONS должна включать топ бар с логотипом и названием команды, навигацию по сайту, фильтрацию по файловой системе (select),
фильтрацию по теме вопроса (select), кнопку добавить вопрос, список вопросов.

По нажатию на “добавить вопрос” должно плавно(с анимацией) открыться модальное окно, в котором будут такие поля:
	- Вопрос - строка, максимум 255 символов
	- Тема - дропдаун с вариантами тем
-  Радио баттоны для выбора ответа ( два варианта - True / False)
-  Чекбоксы для выбора в какой файл записать вопрос ( по дефолту выбран тип файла тот который был выбран в фильтрах) 
	- Кнопка создания вопроса - задизейблена, если нет текста вопроса


После добавления вопроса поля очищаются и модальное окно закрывается, вопрос видно в списке вопросов, если фильтрация по данному типу и теме.
Если вы открыли модальное окно добавления вопросов, достаточно кликнуть за пределами него чтобы оно плавно закрылось, 
так же закрыть его можно с помощью кнопки cancel и крестика 

Список вопросов:
Если вопросов нет, то должно отображаться There are no questions
При выборе файловой системы список вопросов должен фильтроваться по выбранной файловой системе. По дефолту выбран фильтр по файлам JSON
При выборе темы список вопросов должен фильтроваться по выбранной теме. Сортироваться вопросы должны по дате создания вопроса.
Придумать 5 тем вопросов, которые могут быть выбраны для фильтрации и создания( по дефолту выбраны все темы: “all”)

Пример контента вопроса в списке

На каждом вопросе при наведении на обертку должен появиться крестик - кнопка для удаления вопроса, после нажатия на нее,
должно открыться модальное окно с текстом:” Are you sure you want to delete this question? “ И двумя кнопками - confirm and cancel 


Нужно взаимодействовать с файловой системой хранения данных в 4ти форматах (JSON, XML, YAML, CSV) для получения вопросов и записи их


Обязательно хранить состояния значений фильтров в LocalStorage, если мы перезагрузим нашу страницу, выбранные фильтра не должны пропасть. 


Страница About должна содержать текстовую информацию о проекте, например ( Этот проект поддерживает хранение данные в 4 разных форматах ………………………..)  Добавить анимацию на отдельные элементы в тексте например: 
Этот проект поддерживает хранение данные в 4 разных форматах
CSV   JSON   YAML   XML   
Где выделенные строки будут плавно переливаться
